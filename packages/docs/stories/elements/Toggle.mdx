# Toggle

A switch for binary on/off settings. Used for preferences, feature flags, and quick setting changes that take immediate effect.

## When to Use

| Situation | Use Toggle? |
|---|---|
| Binary on/off setting with immediate effect | ✅ Yes |
| Enabling/disabling a feature | ✅ Yes |
| User preferences (notifications, dark mode) | ✅ Yes |
| Form field with save button | ❌ Use Checkbox |
| Selecting one of several options | ❌ Use Radio |
| Confirming action before submit | ❌ Use Checkbox |

**Rule of thumb:** Toggles are for settings that change immediately. Checkboxes are for form inputs that commit on submit.

## Anatomy

```
┌─────────────────────────────────────────┐
│  ┌────────┐   Label                     │
│  │   ●────│                             │
│  └────────┘   Description text          │
│                                         │
│  [1: Track]  [2: Label]                 │
│              [3: Description (optional)] │
│  [Thumb moves left/right on change]     │
└─────────────────────────────────────────┘
```

1. **Track** — Container showing on/off state via color
2. **Thumb** — Moves to indicate state
3. **Label** — Required, describes what the toggle controls
4. **Description** — Optional, explains the consequence

## Accessibility

| Requirement | Implementation |
|---|---|
| Role | `<input type="checkbox" role="switch">` |
| Keyboard | Space to toggle |
| Label | Linked via `htmlFor`/`id` |
| Description | Linked via `aria-describedby` |
| Focus ring | 2px solid, 2px offset |
| Touch target | Entire 44px row is clickable |

### Screen Reader Announcement

```
Default:   "Enable notifications, switch, off"
```

## Do's and Don'ts

✅ **Do**
- Use for settings with immediate effect
- Label clearly with what happens when enabled
- Provide description for consequence-heavy toggles
- Group related toggles together

❌ **Don't**
- Use for form submission (use Checkbox)
- Use ambiguous labels like "Yes/No"
- Put more than one toggle per line of content
- Use for mutually exclusive options (use Radio)

## Code Usage

```tsx
import { Toggle } from '@nirman/components';

// Basic
<Toggle label="Enable notifications" />

// With description
<Toggle
  label="DigiLocker sync"
  description="Automatically store certificates"
  defaultChecked
/>

// Controlled
const [enabled, setEnabled] = useState(false);
<Toggle
  label="Dark mode"
  checked={enabled}
  onChange={(e) => setEnabled(e.target.checked)}
/>
```
