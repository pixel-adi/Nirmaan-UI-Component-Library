# Component Checklist

Every component must pass through these checks before shipping. This is not optional.

## Before You Start Building

| Requirements | Description |
|---|---|
| **Approval** | GitHub Issue explicitly approved with ≥2 team use cases. |
| **Accessibility Spec** | Must define ARIA Roles, Screen Reader Strings, Focus Management, and Keyboard bindings. |
| **Tokens Matrix** | Must define every single variable (`--nir-*`) and Variant Matrix (`variant x size x state`). |

## Figma Component (Designer)

**Figma Variables**
- ✅ Auto-layout for everything (no absolute positioning)
- ✅ ALL fills bound to Semantic/Color variables
- ✅ ALL padding/gap bound to Semantic/Number variables
- ✅ ALL radius bound to variables
- ✅ ZERO hardcoded values anywhere

**Figma Matrices**
- ✅ All `variant × size × state` combinations exist
- ✅ Light mode, Dark mode, High-contrast mode render correctly

**Figma Tests**
- ✅ Script test frames: English, Hindi, Tamil, and RTL renders exist and are unclipped.
- ✅ Accessibility annotation layer added: Role noted, Keyboard shortcuts noted, Touch target dimensions shown, Contrast ratios documented.

## Code Component (Developer)

| Review Phase | Requirements |
|---|---|
| **Architecture** | TypeScript props interface with JSDoc on each prop. `forwardRef` used for composability. |
| **Styles** | CSS uses ONLY `--nir-*` custom properties. BEM naming with `nir-` prefix exactly (`nir-e-button`, `nir-e-button--primary`). |
| **A11y Checks** | `aria-*` attributes match the a11y spec. Keyboard interaction natively implemented. Focus ring visibly works. |
| **Unit Testing** | Renders with correct role, variants apply correct classes, disabled state blocks events, and loading states assert `aria-busy`. |
| **Storybook Docs** | Every variant has an individual story, Size comparisons exist, and an integrated `VernacularTesting` script component exists with autodocs. |

## Documentation (MDX)

The MDX configuration must be robust for external integrators.

- ✅ When to use / When NOT to use
- ✅ Live example with Controls
- ✅ Size guidance
- ✅ Accessibility table
- ✅ Screen reader announcements
- ✅ Do's and Don'ts
- ✅ Code usage example
- ✅ Figma location and component name
- ✅ Related components listed
- ✅ Changelog started

## Audit Score

> [!TIP]
> **Scoring Guidelines**
> Score ≥49/70  →  Ship as ALPHA (internal only)
> Score ≥56/70  →  Ship as BETA (limited adoption)
> Score ≥63/70  →  Ship as STABLE (general availability)

> [!WARNING]
> **CRITICAL BLOCKERS** (any of these completely halts deployment):
> - ✗ ARIA role missing
> - ✗ Keyboard navigation broken
> - ✗ Text contrast below 4.5:1
> - ✗ Any hardcoded value instead of token
> - ✗ Breaks in Hindi or Tamil
