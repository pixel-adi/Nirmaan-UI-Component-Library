# Color System

Nirman uses a **three-tier token architecture** (per Constitution §II.2.1). Primitives are raw values that never appear in components directly. Semantic tokens map primitives to purpose — this is the theming layer. Components consume semantics.

## How Theming Works

Three independent axes control appearance:

| Axis | Controls | Mechanism |
|------|----------|-----------|
| **Brand** | Which org's identity | Overrides `color.primary.*`, `color.accent.*` |
| **Mode** | Light / Dark / HC | Switches `data-theme` attribute on root |
| **Density** | Spacing compactness | Switches `data-density` attribute |

Switch modes using the toolbar above ↑ to see all tokens update live.

## Usage

### In CSS (recommended)
```css
.my-element {
  background: var(--nir-color-primary-default);
  color: var(--nir-color-primary-on-primary);
  border-radius: var(--nir-radius-interactive);
}
```

### In Figma
Apply the variable `color/primary/default` from the **Semantic/Color** collection.
Switch modes (Light → Dark → HC) on any frame to preview.

### Never do this
```css
/* ✗ WRONG — hardcoded value */
.my-element { background: #083459; }

/* ✗ WRONG — primitive token in component */
.my-element { background: var(--nir-color-navy-600); }

/* ✓ CORRECT — semantic token */
.my-element { background: var(--nir-color-primary-default); }
```
