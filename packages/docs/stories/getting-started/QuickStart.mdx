# Quick Start

Build a government service form page in 5 minutes using Nirman components.

## The Goal

A simple "Apply for Income Certificate" form — the most common pattern across all Indian e-District portals.

## Code

```tsx
import '@nirman/tokens/css/primitives';
import '@nirman/tokens/css/light';
import { Button, NirmanProvider } from '@nirman/components';

function IncomeCertificateForm() {
  return (
    <NirmanProvider defaultTheme="light">
      <div style={{
        maxWidth: '600px',
        margin: '0 auto',
        padding: 'var(--nir-spacing-component-padding-lg)',
      }}>
        <h1 style={{
          color: 'var(--nir-color-text-default)',
          fontFamily: 'var(--nir-font-family-body)',
          marginBottom: 'var(--nir-spacing-stack-lg)',
        }}>
          Apply for Income Certificate
        </h1>

        {/* Form fields will go here as we build more components */}

        <div style={{
          display: 'flex',
          gap: 'var(--nir-spacing-inline-md)',
          marginTop: 'var(--nir-spacing-stack-lg)',
        }}>
          <Button variant="primary" type="submit">
            Submit Application
          </Button>
          <Button variant="secondary" type="button">
            Save Draft
          </Button>
        </div>
      </div>
    </NirmanProvider>
  );
}
```

## What's Happening

1. **Token CSS** is imported — all `--nir-*` custom properties are now available globally
2. **NirmanProvider** wraps the app — enables runtime theme/mode switching
3. **Button** uses `variant` prop — primary for main action, secondary for supporting action
4. **All spacing and colors** use `var(--nir-*)` tokens — never hardcoded values
5. **Switch themes** by changing `defaultTheme` to `"dark"` — everything adapts automatically

## Three Theming Axes

```
Brand × Mode × Density = Complete Theme

Brand:    nirman | central-govt | kerala | tamilnadu | up
Mode:     light | dark | high-contrast
Density:  default | compact | spacious
```

Any combination works. A Kerala government portal in dark mode with compact density:

```tsx
<NirmanProvider
  defaultTheme="dark"
  defaultBrand="kerala"
  defaultDensity="compact"
>
  <App />
</NirmanProvider>
```

## Token Hierarchy

```
NEVER USE                    ALWAYS USE
─────────                    ──────────
#083459                  →   var(--nir-color-primary-default)
16px                     →   var(--nir-spacing-component-padding-md)
8px border-radius        →   var(--nir-radius-interactive)
```

Primitives exist but are hidden. Semantic tokens are what you use everywhere — in CSS, in Figma, in code. When the brand changes, semantic tokens resolve to different primitives automatically.
