Accessibility operations in Nirmaan UI occur in **Step 1** of component generation, not via QA testing. By using Nirmaan components, a developer adopts exact Constitutional mandates flawlessly.

## Verifiable Code Parity 

1. **Touch Targets (WCAG 2.5.8):** Minimum `44x44px` bounds are forced inside `Tier 3 (Component)` tokens natively inside inputs and buttons. It is mathematically impossible to produce smaller elements using default `size=md` definitions.
2. **Contrast Levels (WCAG 1.4.3):** Color matrices force a `4.5:1` ratio. Nirmaan inherently offers a `High-Contrast` Theming Mode guaranteeing strict `7:1` (AAA) ratios.
3. **Mandatory ARIA Binding:** Inputs demand `error` states which bind `aria-invalid` implicitly.

> [!IMPORTANT]
> The minimum supported viewport width across every single defined component metric is bounded strictly to **320px**. All components break down fluidly without horizontal scroll.
