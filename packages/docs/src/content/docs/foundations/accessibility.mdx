Accessibility in Nirmaan UI occurs in **Step 1** of component generation — never as a QA afterthought. Using Nirmaan components means natively inheriting exact Constitutional and WCAG 2.1 AA mandates. Engineering teams cannot accidentally violate accessibility because those errors are mathematically nullified at the token layer.

## Touch Targets (WCAG 2.5.8)

| Size | Height | Meets WCAG 2.5.8? | Use when |
| :--- | :--- | :--- | :--- |
| Small (`sm`) | 36px | ❌ No | Compact/admin interfaces only |
| **Medium (`md`)** | **44px** | **✅ Yes — Default** | **All public-facing forms** |
| Large (`lg`) | 52px | ✅ Yes | High-impact primary actions |

> [!IMPORTANT]
> Always use `size="md"` for public-facing forms. `sm` size exists only for compact admin UIs where density is critical.

## Focus Indicators

Minimum 2px outline, 2px offset, must contrast 3:1 against adjacent background.

```css
.nir-e-button:focus-visible {
  outline: 2px solid var(--nir-colour-primary-base);
  outline-offset: 2px;
}
```

> [!TIP]
> Use `:focus-visible` — not `:focus`. This prevents outlines on mouse clicks while maintaining them for keyboard navigation.

## Keyboard Navigation

| Component | Key | Action |
| :--- | :--- | :--- |
| Button | `Enter`, `Space` | Activate |
| Input | `Tab` | Focus |
| Checkbox | `Space` | Toggle |
| Radio | `Arrow keys` | Navigate group |
| Toggle | `Space` | Switch on/off |
| Dropdown | `↑↓`, `Enter`, `Escape` | Navigate, select, close |
| Autocomplete | Type, `↑↓`, `Enter` | Filter, navigate, select |

## ARIA Attributes

| Component | Role | Key attributes |
| :--- | :--- | :--- |
| Button | `button` (native) | `aria-disabled`, `aria-busy` |
| Input | `textbox` (native) | `aria-invalid`, `aria-describedby` |
| Checkbox | `checkbox` (native) | `aria-checked`, `aria-describedby` |
| Radio | `radio` (native) | `aria-checked`, wrapped in `radiogroup` |
| Toggle | `switch` | `aria-checked` |
| Dropdown | `combobox` | `aria-expanded`, `aria-activedescendant` |
| Error message | — | `role="alert"` |

## Multi-Script Accessibility

| Concern | Solution |
| :--- | :--- |
| Devanagari text clips | Line-height 1.7 (vs 1.5 Latin) |
| Tamil/Malayalam dense conjuncts | `+0.01em` letter-spacing |
| Urdu RTL direction | `direction: rtl` via `:lang(ur)` |
| Screen reader pronunciation | Always set `lang` attribute |

### The `lang` Attribute is Critical

```html
<!-- Correct — screen reader reads with Hindi pronunciation -->
<p lang="hi">आवेदन जमा करें</p>

<!-- WRONG — screen reader reads Hindi with English pronunciation rules -->
<p>आवेदन जमा करें</p>
```

## Disabled States

Use `aria-disabled="true"` rather than HTML `disabled` when possible. HTML `disabled` removes the element from tab order entirely — users then lose awareness of its existence.

```css
/* Consistent disabled visual across all components */
.nir-e-button--disabled {
  opacity: var(--nir-opacity-disabled); /* 0.38 */
  cursor: not-allowed;
}
```
