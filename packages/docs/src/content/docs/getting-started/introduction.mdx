This documentation defines the **Nirmaan UI Design System**. It exists to solve a specific problem: the absence of a unified, enforceable design language for digital public infrastructure in India — one that is simultaneously accessible to a user in Vidarbha on a 2G network, and a bureaucrat in New Delhi on a desktop.

It is not a component library. It is not a UI kit. It is a **constitutional framework for design-code parity**.

## The Six Principles

Every architectural decision and component specification traces its reasoning back to these immovable philosophical foundations:

1. **Janabhasha (जनभाषा) — The People's Language**
   Every interface must communicate through typographic respect for each script's unique visual grammar. Nirmaan utilizes a script-aware token architecture scaling 12+ scripts inherently.

2. **Sarvajanik (सार्वजनिक) — Universal by Default**
   Accessibility is the foundational layer. Minimum contrast ratios (4.5:1) and touch target minimums (44×44px) are mathematically bound at the token level.

3. **Parivartan-sheelta (परिवर्तनशीलता) — Radical Adaptability**
   Any organization must be able to express their identity without forking the codebase. Our three-tier architecture achieves this.

4. **Vishvasaniya (विश्वसनीय) — Trustworthy Consistency**
   Figma logic and engineering logic must explicitly maintain verifiable parity. Component APIs are contractual.

5. **Saral (सरल) — Complexity Absorbed**
   The system must function securely at 320px viewport widths and load seamlessly.

6. **Surakshit (सुरक्षित) — Secure by Pattern**
   Interaction design is secure by default (masking, OTP inputs, session timeouts).
