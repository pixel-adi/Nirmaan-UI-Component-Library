Developers consuming the Nirmaan UI architecture do not overwrite configurations or "theme" component wrappers arbitrarily. They operate explicitly in **Layer 5 (Pages)** and **Layer 4 (Templates)**. 

Components are guaranteed contractual APIs adhering to **Vishvasaniya (Trustworthy Consistency)** principles.

## The Five-Layer Taxonomy

You consume modules from these explicitly defined structures:

1. **Layer 1: Elements (Atoms):** The smallest reusable units. Single interactions (`<Button>`, `<Input>`).
2. **Layer 2: Patterns (Molecules):** Functional groupings (`<Search Bar>`, `<Form Field>`).
3. **Layer 3: Blocks (Organisms):** Self-contained complex regions (`<Government Header>`, `<Data Table>`).
4. **Layer 4: Templates:** Spatial layouts scaling components rapidly.
5. **Layer 5: Pages:** Real content implementations.
