Nirmaan UI designers do not arbitrarily draw elements. Every component, regardless of layer, follows the **Component Construction Protocol** enforced universally.

## Component Construction Protocol

**Step 1 — Accessibility specification (Written First)**
Before any visual design, document the component's ARIA contract, keyboard model, and screen reader announcements.

**Step 2 — Token binding**
Create or reference the component's **Tier 3 (Component) tokens** via Token Studio for Figma. The property must explicitly bind to a **Tier 2 (Semantic) token**. Zero hardcoded values.

**Step 3 — Base component construction**
Build using Figma's auto-layout. All spacing, padding, and sizing strictly references Native spacing variables. Naming must follow the `jds-[layer]-[name]-[variant]--[state]` convention.

**Step 4 — Variant matrix**
Define every visual modification. Every combination of variant × size × state must exist explicitly in the properties panel.

**Step 5 — Accessibility annotation layer**
Annotate the component showing ARIA roles, touch target dimensions (44x44 minimum), and contrast ratios.

**Step 6 — Multi-script validation**
Place the element in test frames mapping Hindi, Tamil, Telugu, and Urdu strings. Verify zero clipping occurs mathematically before committing the token definitions to GitHub!
