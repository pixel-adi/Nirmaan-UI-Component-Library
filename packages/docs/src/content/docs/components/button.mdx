Buttons are Layer 1 interactive Elements. Nirmaan Button constraints explicitly map to Constitutional Accessibility standards. 

## 1. Accessibility Specification
- **ARIA:** Requires explicit `aria-label` or implicit text content guaranteeing readable identifiers.
- **Keyboard:** Triggers actively on `SPACE/ENTER` keys natively.
- **Min Touch Bounds:** The default bounds are governed strictly by the component token `min-height: 44px`.

## 2. Token Binding
- **Backgrounds:** Bind directly to `color.primary.default`.
- **Text:** Bind cleanly to `color.text.inverse`.
- **States:** Hover binds to `color.primary.hover` securely handling pseudo-states independently.

## 4. Engineering Implementation

The semantic properties natively handle states locally across platforms.

  <TabItem label="React Usage">
    ```tsx
    import { Button } from '@nirman/components';
    ```

  </TabItem>
  <TabItem label="SwiftUI Usage">
    ```swift
    Button(action: submit) {
        Text("Submit Application")
    }
    .background(Tokens.Color.Primary.default)
    .cornerRadius(Tokens.Radius.interactive)
    .frame(minHeight: Tokens.Spacing.touchTarget) // explicitly applies 44px
    ```
  </TabItem>
  <TabItem label="Jetpack Compose">
    ```kotlin
    Button(
        onClick = { submit() },
        colors = ButtonDefaults.buttonColors(
            backgroundColor = Tokens.ColorPrimaryDefault
        ),
        modifier = Modifier.defaultMinSize(minHeight = Tokens.SpacingTouchTarget)
    ) {
        Text("Submit Application", color = Tokens.ColorTextInverse)
    }
    ```
  </TabItem>
