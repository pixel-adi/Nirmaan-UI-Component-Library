Typography in Nirmaan UI is handled natively via explicit CSS utility classes injected by `@nirman/tokens`. It is a structural engine that maps properties directly to HTML elements.

## Interactive Playground

<Tabs>
  <TabItem label="Web Usage (React/HTML)">
    ```tsx
    // Any HTML element can adopt the styling scale seamlessly
    <h1 className="nir-type-h2">
      Accessible DOM Title
    </h1>
    ```
  </TabItem>
  <TabItem label="Native Implementations">
    The multi-script overrides (via `:lang()`) don't apply mobile natively unless built out. Mobile engineers just use the generated Composite properties:
    ```swift
    // iOS
    Text("Heading")
     .font(Tokens.Composite.HeadingMd) 
    ```
  </TabItem>
</Tabs>
